# 微信连接 Claude Code 使用指南

## 工作原理

```
你的手机微信 ←→ ilink bot API（微信官方） ←→ 本机 Node.js 守护进程 ←→ Claude Code SDK
```

扫码后，微信会把你的账号绑定为一个 ilink bot，守护进程通过长轮询监听消息，转发给本机的 Claude Code 执行。

---

## 前置条件

| 条件 | 说明 |
|------|------|
| Node.js >= 18 | `node -v` 确认版本 |
| Claude Code 已安装 | `claude --version` 确认可用 |
| ANTHROPIC_API_KEY | 已配置到环境变量或 Claude Code 登录态 |
| 个人微信账号 | 用于扫码绑定，**不支持企业微信** |
| macOS 或 Linux | Windows 暂不支持守护进程模式 |

---

## 第一步：安装项目

将当前项目链接到 Claude Code 的 skills 目录：

```bash
# 创建 skills 目录（如果不存在）
mkdir -p ~/.claude/skills

# 软链接到 skills 目录，这样 /wechat 命令可用
ln -s "$(pwd)" ~/.claude/skills/wechat-claude-code
```

然后安装依赖并编译：

```bash
npm install
```

`npm install` 会自动执行 `tsc` 编译，完成后 `dist/` 目录会生成编译产物。

---

## 第二步：扫码绑定微信

```bash
# 在当前项目目录下执行
npm run setup
```

执行后会自动弹出二维码图片（macOS/Linux GUI），或在终端内显示二维码（无界面 Linux）。

**用微信扫码步骤：**
1. 打开微信 → 扫一扫
2. 扫描弹出的二维码
3. 在手机上点击「确认登录」
4. 终端显示 `✅ 绑定成功!` 即完成

扫码成功后，程序会询问工作目录：

```
请输入工作目录 [/当前路径]:
```

输入你希望 Claude 操作的项目路径（直接回车使用当前目录）。

> 绑定信息保存在 `~/.wechat-claude-code/accounts/` 下，下次启动无需重新扫码。

---

## 第三步：启动守护进程

```bash
npm run daemon -- start
```

- **macOS**：注册 launchd，开机自启，崩溃自动重启
- **Linux（有 systemd）**：注册 systemd user service
- **Linux（无 systemd）**：nohup 后台运行

启动成功后终端输出：

```
Started wechat-claude-code daemon (macOS launchd)
```

---

## 第四步：在微信中发消息

直接在微信中给**自己**发消息（文字或图片），Claude 会自动回复。

> 注意：消息发给自己（文件传输助手不行），因为绑定的是你自己的账号作为 bot。

---

## 常用管理命令

```bash
npm run daemon -- status    # 查看运行状态
npm run daemon -- stop      # 停止服务
npm run daemon -- restart   # 重启（更新代码后使用）
npm run daemon -- logs      # 查看最近日志
```

## 微信端可用命令

| 命令 | 说明 |
|------|------|
| `/help` | 显示所有命令 |
| `/status` | 查看当前会话状态（工作目录、模型、权限模式） |
| `/cwd <路径>` | 切换 Claude 的工作目录 |
| `/model <名称>` | 切换模型，如 `/model claude-sonnet-4-6` |
| `/permission <模式>` | 切换权限模式（见下方说明） |
| `/prompt <内容>` | 设置全局系统提示词，如 `/prompt 用中文回答` |
| `/prompt clear` | 清除系统提示词 |
| `/clear` | 清除当前会话（重新开始） |
| `/compact` | 压缩上下文（token 清零，保留历史记录） |
| `/history [N]` | 查看最近 N 条对话记录 |
| `/undo [N]` | 撤销最近 N 条对话 |
| `/skills` | 列出已安装的 Claude Code Skill |
| `/reset` | 完全重置（包括工作目录等所有设置） |

## 权限模式说明

当 Claude 需要执行工具（读写文件、运行命令等）时，会发消息到微信请求审批。

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `default` | 每次工具调用都需要手动回复 y/n | 日常使用，最安全 |
| `acceptEdits` | 文件编辑自动批准，其他需审批 | 写代码时减少打断 |
| `plan` | 只读，不允许任何写操作 | 只想让 Claude 分析代码 |
| `auto` | 全自动，无需审批（需二次确认） | 批量任务，风险自担 |

切换示例：
```
/permission acceptEdits
```

开启 auto 模式需要二步确认：
```
/permission auto
# 收到警告后再发：
/permission auto confirm
```

---

## 权限审批操作

Claude 请求工具时，微信会收到如下消息：

```
🔧 权限请求

工具: Bash
输入: npm install

回复 y 允许，n 拒绝
(120秒未回复自动拒绝)
```

- 回复 `y` 或 `yes` → 允许
- 回复 `n` 或 `no` → 拒绝
- 120 秒内不回复 → 自动拒绝

## 数据目录结构

```
~/.wechat-claude-code/
├── accounts/           # 微信账号凭证（含 botToken，权限 600）
├── config.env          # 全局配置（工作目录、模型、权限模式、系统提示词）
├── sessions/           # 会话数据（含对话历史）
├── get_updates_buf     # 消息轮询同步缓冲
└── logs/               # 运行日志（stdout.log / stderr.log）
```

## 常见问题

**Q：扫码后提示「不支持此版本」**
微信版本过低，更新到最新版本后重试。

**Q：消息发出去没有回复**
1. `npm run daemon -- status` 确认服务在运行
2. `npm run daemon -- logs` 查看错误日志
3. 确认 `ANTHROPIC_API_KEY` 环境变量已设置

**Q：服务启动后 Claude 报错「找不到 claude 命令」**
确保 Claude Code CLI 已安装且在 PATH 中：`which claude`

**Q：想换一个微信账号**
删除 `~/.wechat-claude-code/accounts/` 下的 JSON 文件，重新运行 `npm run setup`。

**Q：工作目录切换失败**
`/cwd` 只接受已存在的绝对路径，且不能是系统目录（`/etc`、`/sys` 等）。

## 环境变量（可选）

如果使用第三方 API 或自定义端点，在启动前设置：

```bash
export ANTHROPIC_API_KEY=your_key
export ANTHROPIC_BASE_URL=https://your-proxy.com  # 可选，使用代理
```

macOS launchd 会在 `daemon start` 时自动读取当前 shell 的这些变量写入 plist。
